2
Admin → Backend → MongoDB

admin สร้างบัตร

admin สร้างรหัสบัตร (กำหนดจำนวนวัน) เก็บใน cards สถานะ unused

4

3) Flow B — ดูหนัง (หัวใจของระบบ) 🔑

ขั้นที่ไฮไลต์สีส้ม = งานของ video-worker

1
Frontend → Backend

กดเล่น → ขอ “ตั๋ว”

POST /playback/start → backend เช็ค บัตร active? + concurrency (1 เครื่อง)? → ถ้าผ่าน เซ็น grant (HMAC) แล้วคืน authUrl + fileUrl

2
Frontend → video-worker

เอาตั๋วไปแลก cookie

frontend fetch(authUrl) → worker ตรวจลายเซ็น+วันหมดอายุ+IP ของ grant → ถ้าผ่าน Set-Cookie (HttpOnly, อายุสั้น ~2 นาที)

3
<video> → video-worker → R2

โหลด/เล่นไฟล์ (Range)

<video src=fileUrl> ยิง Range request พร้อม cookie อัตโนมัติ → worker ตรวจ cookie + เช็คว่าตั๋วตรงกับไฟล์ที่ขอ → ดึงช่วง byte จาก R2 → ส่ง 206 Partial Content → เล่น/seek ได้

4
Frontend → Backend → video-worker

ต่ออายุ (heartbeat) ทุก ~60 วิ

POST /playback/refresh → grant ใหม่ → set cookie ใหม่ (เนียน ไม่ต้องเปลี่ยน src) · ถ้าเครื่องอื่นมาเล่นบัญชีเดียวกัน → เครื่องเก่า refresh ไม่ผ่าน (409) = ถูกเตะ

5
ออกจากหน้า

ปิดสตรีม

POST /playback/stop เคลียร์สถานะ · cookie หมดอายุเองใน ~2 นาที

4) เจาะลึก video-worker — ทำ 2 งาน

GET /__auth?token=…

  • อ่าน grant จาก query
  • คำนวณ HMAC ใหม่ด้วย VIDEO_GRANT_SECRET แล้วเทียบลายเซ็น (constant-time)
  • เช็ค exp ยังไม่หมด
  • เช็ค IP (ถ้าเปิด ENFORCE_IP)
  • ถ้าผ่าน → Set-Cookie mdn_video (HttpOnly·Secure·SameSite=None) อายุ = เวลาที่เหลือของ grant

GET /<r2Key> (ไฟล์)

  • อ่าน grant จาก cookie → verify เหมือนข้างซ้าย
  • เช็ค grant.key === ไฟล์ที่ขอ (กันเอาตั๋วเรื่อง A ไปดึงเรื่อง B)
  • เช็ค IP (ถ้าเปิด)
  • แปลง Range: → ดึงเฉพาะช่วง byte จาก R2 binding
  • ส่ง 206 + Content-Range (รองรับ seek) — bytes วิ่ง R2→client ตรง

คล้าย JWT แบบย่อ: ข้อมูล + ลายเซ็น HMAC-SHA256 เซ็นด้วย secret ที่ backend & worker รู้ร่วมกันเท่านั้น

6) ทำไมถึงปลอดภัย & ประหยัด

✅ ระบบนี้กันได้

  • egress ฟรี: ไฟล์วิ่ง R2→worker→client บนเครือข่าย Cloudflare ไม่ผ่าน backend
  • backend ไม่ถือ key ของ R2: worker เข้าผ่าน binding → backend ถูกเจาะก็เข้า storage ไม่ได้
  • ลิงก์แชร์/hotlink: ตั๋วอายุสั้น + ผูก IP + ผูกไฟล์เดียว
  • แชร์บัญชี: 1 บัญชี = 1 สตรีม (เปิดเครื่องใหม่เตะเครื่องเก่า)

⚠️ สิ่งที่กันไม่ได้ 100%

  • คนที่จ่ายเงิน+login แล้ว ยังใช้เครื่องมือ (เช่น yt-dlp) ดูดไฟล์ของตัวเองได้ — เป็นธรรมชาติของ MP4 ในเบราว์เซอร์
  • ถ้าต้องกัน rip จริงจัง = ต้องใช้ DRM (Widevine/FairPlay) ซึ่งหนัก/แพง — เก็บไว้เป็นเฟสอนาคต
